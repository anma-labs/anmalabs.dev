The Problem

Every new session, your AI agent re-reads everything.

Every time you open Claude Code on an existing project, it scans your source files, infers interfaces, and guesses at conventions before writing a single line of code.

ANMA inverts this. Instead of dumping your codebase into context, you write contracts that declare your architecture in ~350 tokens per module. The AI reads contracts and starts building immediately — knowing every interface, dependency, error type, and behavioral invariant.

How It Works

Define. Validate. Build.

Define contracts

Each module gets a CONTRACT.yaml — inputs, outputs, error codes, and behavioral invariants. ~350 tokens per module. This is what your AI agent reads instead of source code.

Validate with the linter

24 structural checks plus 7 architectural principles catch undeclared dependencies, missing error codes, gateway violations, and naming issues — before implementation starts.

Build from contracts

Point Claude Code at your contracts. It implements interfaces exactly as specified. Contracts persist across sessions — no re-explaining your architecture.

Architecture as Data

Any agent. Any session. Zero ramp-up.

A new AI agent reads these files and has full context to continue building. No onboarding, no "let me scan the codebase," no hallucinated interfaces.

~90 tokens
